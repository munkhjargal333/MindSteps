# Сэтгэцийн эрүүл мэндийн систем

## 1. Төслийн танилцуулга

Энэхүү төсөл нь хэрэглэгчийн сэтгэл зүйн байдлыг өдөр тутам тэмдэглэх, сэтгэл хөдлөлөө ажиглах, өөрийн үнэт зүйлсээ тодорхойлох замаар өөрийгөө ойлгох, сэтгэцийн эрүүл мэнддээ анхаарах боломж олгох веб систем юм.

## 2. Багийн гишүүд

* **Ц. Мөнхжаргал** – Хөгжүүлэгч

## 3. Суулгах заавар

### Backend

```bash
cd backend
go mod tidy
# өгөгдлийн сангаас model үүсгэх
go run cmd/gen/main.go
# backend сервер ажиллуулах
go run cmd/api/main.go
# unit test ажиллуулах
go test ./test/unit/...
```

### Frontend

```bash
cd frontend
yarn install
yarn run dev
#integration test ажиллуулах
yarn test
```

## 4. Хийсэн зүйлс

### 4.1 Технологийн стек

* **Backend:** Golang (Fiber)
* **Frontend:** Next.js, Tailwind CSS
* **Өгөгдлийн сан:** Neon (PostgreSQL)
* **Deployment:** Render, Vercel

### 4.2 Реализацлагдсан гол функцууд

* Хэрэглэгчийн бүртгэл, нэвтрэх систем
* Өдрийн тэмдэглэл бүртгэл
* Үнэт зүйл бүртгэл
* Сэтгэл хөдлөлийн бүртгэл
* Зорилго үүсгэх, удирдах
* Бясалгал (demo хувилбар)

### 4.3 Аюулгүй байдал

* SQL Injection халдлагаас сэргийлсэн *(done)*
* XSS халдлагаас хамгаалалт *(not yet)*
* Нууц үгийг bcrypt ашиглан хэшлэх *(done)*
* JWT токен дээр суурилсан баталгаажуулалт *(done)*

## 5. Тулгарсан бэрхшээлүүд

### 5.1 Техникийн бэрхшээл

* `.env` тохиргооноос шалтгаалсан датабазын холболт тасалдах асуудал
* Migration зөрөх, schema өөрчлөгдөхөд алдаа гарах
* CORS тохиргооны алдаа
* Build амжилтгүй болох (Next.js / Go version зөрөх)
* Render, Vercel дээр environment variable өөрчлөх

### 5.2 Багийн ажиллагааны бэрхшээл
* Git merge conflict

### 5.3 Цагийн менежментийн бэрхшээл
* Урьдчилан тооцоолсноос илүү хугацаа зарцуулагдсан
* Шаардлагын гэнэтийн өөрчлөлтүүд

## 6. Сурсан зүйлс


### 6.1 Техникийн мэдлэг

* Fiber framework-ийг илүү гүнзгий түвшинд ойлгосон
* Unit/integration test бичих дадал эзэмшсэн
* REST API дизайн хийх ойлголтоо сайжруулсан
* Аюулгүй байдлын суурь ойлголтууд
* Test бичих боломжтой код бүтэц үүсгэх
* CI/CD ашиглах хянах
* Lint ашиглан кодын чанарыг сайжруулсан

### 6.2 Багийн ажиллагаа

* Git ашиглан хамтын хөгжүүлэлт хийх
* Код ревью хийх арга барил

### 6.3 Төслийн удирдлага

* Хугацааны төлөвлөлт хийх, баримтлах
* Даалгавар хуваарилах
* Ахиц дэвшлийг хэмжих аргачлал

## 7. Ирээдүйн төлөвлөгөө

1. Тэмдэглэлийн өгөгдөлд анализ хийх модуль нэмэх
2. Frontend-ийн UI/UX загварыг сайжруулах
3. Сэтгэл хөдлөлийн бүртгэл хувиргах (emotion transformation)
4. Ишлэл болон зохиомжийн хэсгийг хөгжүүлэх
5. Хэрэглэгч бүрт тохирсон AI-д суурилсан meditation судлах

## 8. Дүгнэлт

Энэхүү төсөл нь сэтгэцийн эрүүл мэндийн чиглэлд технологийг ашиглан бодит хэрэгцээг шийдэх боломжтойг харуулсан туршилт байлаа. Хөгжүүлэлтийн явцад техникийн болон багийн ажиллагааны олон сорилтууд тулгарсан ч эдгээр нь цаашдын төслүүдэд илүү чанартай, төлөвлөгөөтэй ажиллах суурь туршлага болж өгсөн. Цаашид уг системийг илүү ухаалаг, хэрэглэгч төвтэй болгон хөгжүүлэх бүрэн боломжтой гэж дүгнэж байна.
